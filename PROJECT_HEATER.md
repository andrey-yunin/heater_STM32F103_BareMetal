# Проект: Интеллектуальный контроллер ИК-обогревателя (Smart Heater)

## 1. Цель проекта
Создание отказоустойчивой системы управления обогревом с удаленным контролем через Android-приложение, разделяющей задачи безопасности и связи между двумя микроконтроллерами.

## 2. Системная архитектура (Hybrid Architecture)
Система строится на базе двух узлов, общающихся по интерфейсу UART:

### Узел А: STM32F103 (Core Control - Bare Metal)
*   **Роль:** «Мозг» и исполнительный механизм.
*   **Задачи:**
    *   Опрос датчика температуры (DS18B20) по протоколу OneWire.
    *   Реализация алгоритма термостата (гистерезис или ПИД-регулирование).
    *   Управление силовым реле через GPIO.
    *   Обеспечение аппаратной безопасности (Watchdog, Fail-safe).
    *   Прием команд от ESP32 и отправка телеметрии по UART.
*   **Среда:** STM32CubeIDE (Bare Metal C / CMSIS).

### Узел Б: ESP32 (Communication Gateway)
*   **Роль:** «Мост» между интернетом и физическим миром.
*   **Задачи:**
    *   Подключение к Wi-Fi (Station Mode).
    *   Запуск локального HTTP-сервера.
    *   Обработка запросов от Android-приложения.
    *   Передача уставок (целевая температура) в STM32 по UART.
    *   Получение данных от STM32 для отображения в приложении.
*   **Среда:** VS Code + PlatformIO (ESP-IDF Framework).

---

## 3. Схема взаимодействия (Протокол UART)
Обмен данными производится текстовыми пакетами (например, JSON или упрощенный формат):
*   **STM32 -> ESP32:** `{"temp": 22.5, "relay": 1, "status": "OK"}` (раз в 2 сек).
*   **ESP32 -> STM32:** `{"set_temp": 24.0}` (только при изменении).

## 4. План реализации (Дорожная карта)

### Фаза 1: STM32 Bare Metal (Нижний уровень)
1.  **[x] Hello World:** Настройка тактирования (RCC) и мигание светодиодом (GPIO) через регистры. (ВЫПОЛНЕНО)
2.  **[x] UART Driver:** Написание драйвера UART (прерывания NVIC, Loopback-тест пройден). (ВЫПОЛНЕНО)
3.  **[ ] ADC Lab:** Чтение аналогового сигнала (подготовка к датчику температуры).
4.  **[ ] Sensor Lab:** Реализация протокола OneWire и получение данных с DS18B20.
5.  **[ ] Logic:** Алгоритм включения реле при падении температуры.

### Фаза 2: ESP32 Connectivity (Сетевой уровень)
1.  **[ ] Wi-Fi Init:** Подключение к домашней сети.
2.  **[ ] HTTP Server:** Создание эндпоинтов /status и /set.
3.  **[ ] Bridge:** Реализация UART-обмена с STM32.

### Фаза 3: Android Application
1.  **[ ] UI Design:** Экран с текущей температурой и ползунком.
2.  **[ ] API Integration:** Отправка HTTP-запросов на ESP32.

### Фаза 4: Безопасность и Сборка
1.  **[ ] Watchdog:** Настройка независимого сторожевого таймера.
2.  **[ ] Error Handling:** Алгоритм отключения реле при обрыве связи или датчика.
3.  **[ ] Final Assembly:** Монтаж в корпус и тестирование под нагрузкой.

---

## 5. Спецификация компонентов (BOM)
1.  Микроконтроллер STM32F103C8T6 (Blue Pill).
2.  Микроконтроллер ESP32 (WROOM-32).
3.  Программатор ST-Link V2 (для STM32).
4.  Датчик температуры DS18B20 (рекомендуется в гильзе).
5.  Модуль реле (5V/10A с опторазвязкой, совместимый с 3.3V логикой).
6.  Блок питания 5V (минимум 1A).
