# Проект: Интеллектуальный контроллер ИК-обогревателя (Smart Heater)

## 1. Цель проекта
Создание отказоустойчивой модульной системы управления обогревом с удаленным контролем.

## 2. Целевая архитектура (Layered Modular Design)
Система разделена на независимые слои для обеспечения возможности замены модулей без переписывания логики.

### 2.1 Layer 0: Hardware Abstraction Layer (HAL / Drivers)
*Прямая работа с регистрами. Файлы:*
*   `stm32f103_regs.h`: Единая база адресов всех используемых регистров.
*   `rcc.c/h`: Настройка тактирования (PLL 72MHz) и системного таймера SysTick.
*   `gpio.c/h`: Унифицированное управление пинами (Реле, Светодиоды, Кнопки).
*   `uart.c/h`: Драйвер USART1 (прерывания, передача байт/строк).
*   `adc.c/h`: Драйвер АЦП (чтение температуры чипа и внешних каналов).
*   `onewire.c/h`: Реализация протокола OneWire для датчика DS18B20.

### 2.2 Layer 1: System Services (Middleware)
*Логика обмена и системные службы. Файлы:*
*   `protocol.c/h`: Машина состояний (State Machine) для разбора бинарных пакетов v1.2, расчет CRC8.
*   `sys_time.c/h`: Служба системного времени (Uptime, программные таймеры).

### 2.3 Layer 2: Application Logic (Business Logic)
*Управление прибором. Файлы:*
*   `thermostat.c/h`: Алгоритм поддержания температуры, работа с уставками (Target Temp).
*   `safety.c/h`: Модуль безопасности (Watchdog, аварийное отключение при перегреве/отказе датчика).
*   `main.c`: Точка входа, инициализация слоев и главный цикл синхронизации.

---

## 3. Схема взаимодействия (Hybrid Architecture)
*   **STM32F103:** Сбор данных, логика термостата, силовое управление.
*   **ESP32:** Связь с Wi-Fi, обработка HTTP-запросов, проброс команд в STM32 по UART.

## 4. План реализации (Roadmap)

### Фаза 1: Проверка концепции (ВЫПОЛНЕНО)
1.  **[x] Мигание светодиодом через регистры.**
2.  **[x] Базовая работа UART на прерываниях.**
3.  **[x] Тестирование Loopback (передача/прием символов).**
4.  **[x] Разработка спецификации протокола v1.2.**

### Фаза 1.1: Реализация модульной архитектуры (ВЫПОЛНЕНО)
1.  **[x] Infrastructure:** Создание `stm32f103_regs.h`.
2.  **[x] Driver Layer:** Модули `rcc.c/h`, `uart.c/h`, `sys_time.c/h`.
3.  **[x] GPIO Module:** Модуль управления реле и индикацией.
4.  **[x] Архитектурный рефакторинг:** Разделение на слои и перенос логики в App/.

### Фаза 1.2: Датчики и Логика (ТЕКУЩИЙ ЭТАП)
1.  **[>] Transport Layer:** Кольцевой буфер и API для сырых байт [В ПРОЦЕССЕ].
2.  **[ ] Protocol Engine:** Сборщик фреймов v1.2 ($ + XOR).
3.  **[ ] ADC Module:** Мониторинг напряжения и температуры MCU.
4.  **[ ] DS18B20:** Драйвер OneWire (Bare Metal).
5.  **[ ] Thermostat:** Основная логика управления.

### Фаза 1.2: Датчики и Логика
1.  **[ ] ADC Module:** Настройка АЦП для мониторинга напряжения и температуры чипа.
2.  **[ ] DS18B20:** Написание драйвера OneWire (Bare Metal тайминги).
3.  **[ ] Thermostat:** Реализация логики управления реле на основе данных датчика.

### Фаза 2: ESP32 Connectivity
1.  **[ ] Wi-Fi Init:** Настройка подключения к сети в режиме Station.
2.  **[ ] HTTP Server:** Создание API для управления (GET /status, POST /set).
3.  **[ ] Bridge:** Синхронизация данных с STM32 по бинарному протоколу.

### Фаза 3: Android Application
1.  **[ ] UI Design:** Разработка интерфейса (Температура, График, Уставки).
2.  **[ ] API Integration:** Реализация обмена данными с ESP32.

### Фаза 4: Безопасность и Сборка
1.  **[ ] Watchdog:** Настройка аппаратного сторожевого таймера.
2.  **[ ] Error Logic:** Обработка обрыва связи между контроллерами.
3.  **[ ] Assembly:** Финальный монтаж в корпус и тесты под нагрузкой.

## 5. Спецификация компонентов (BOM)
1.  STM32F103C8T6 (Blue Pill).
2.  ESP32 (WROOM-32).
3.  ST-Link V2.
4.  Датчик DS18B20 (в герметичной гильзе).
5.  Модуль реле (5V/10A с опторазвязкой, 3.3V логика).
6.  Блок питания 5V (минимум 2A).
