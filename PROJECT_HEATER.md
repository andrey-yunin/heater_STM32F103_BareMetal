# Проект: Интеллектуальный контроллер ИК-обогревателя (Smart Heater)

## 1. Цель проекта
Создание отказоустойчивой модульной системы управления обогревом с удаленным контролем.

## 2. Целевая архитектура (Layered Modular Design)
Система разделена на независимые слои для обеспечения возможности замены модулей без переписывания логики.

### 2.1 Layer 0: Hardware Abstraction Layer (HAL / Drivers)
*Прямая работа с регистрами. Файлы:*
*   `stm32f103_regs.h`: Единая база адресов всех используемых регистров.
*   `rcc.c/h`: Настройка тактирования (PLL 72MHz) и системного таймера SysTick.
*   `gpio.c/h`: Унифицированное управление пинами (Реле, Светодиоды, Кнопки).
*   `uart.c/h`: Драйвер USART1 (прерывания, передача байт/строк).
*   `adc.c/h`: Драйвер АЦП (чтение температуры чипа и внешних каналов).
*   `onewire.c/h`: Реализация протокола OneWire для датчика DS18B20.

### 2.2 Layer 1: System Services (Middleware)
*Логика обмена и системные службы. Файлы:*
*   `fifo_handler.c/h`: Кольцевой буфер для сырых байт.
*   `frame_packer.c/h`: Сборщик байтов из FIFO во фрейм v1.2 ($ + XOR).
*   `frame_parser.c/h`: Интерпретатор смысла фреймов (Команды/Данные).
*   `sys_time.c/h`: Служба системного времени (Uptime, программные таймеры).

### 2.3 Layer 2: Application Logic (Business Logic)
*Управление прибором. Файлы:*
*   `cmd_handler.c/h`: Распределитель команд (Bridge между связью и логикой).
*   `thermostat.c/h`: Автономный алгоритм поддержания температуры (Master STM32).
*   `safety.c/h`: Модуль безопасности (Watchdog, аварийное отключение, мониторинг связи).
*   `main.c`: Точка входа, инициализация слоев и главный цикл синхронизации.

### 2.4 Layer 3: Application Services (Orchestration)
*Высокоуровневые задачи, управляющие логикой и связью. Файлы:*
*   `telemetry_manager.c/h`: Управление периодическими и событийными отчетами (Heartbeat 30с).
*   `thermostat_manager.c/h`: Координация автономного термостата.
*   `system_init.c/h`: Унифицированная инициализация всех подсистем.
*   `main.c`: Точка входа, оркестрация неблокирующих Task-функций.

---

## 3. Схема взаимодействия (Hybrid Architecture)
*   **STM32F103 (Master Logic):** Сбор данных, автономный термостат, управление реле.
*   **ESP32 (User Interface Bridge):** Связь с Wi-Fi, API для мобильного приложения, проброс команд.

### Фаза 1.1: Реализация модульной архитектуры (ВЫПОЛНЕНО)
1.  **[x] Infrastructure:** Создание `stm32f103_regs.h`.
2.  **[x] Driver Layer:** Модули `rcc.c/h`, `uart.c/h`, `sys_time.c/h`.
3.  **[x] GPIO Module:** Модуль управления реле и индикацией.
4.  **[x] Архитектурный рефакторинг:** Разделение на слои и перенос логики в App/.

### Фаза 1.2: Датчики и Логика (ВЫПОЛНЕНО)
1.  **[x] FIFO Handler:** Модуль кольцевого буфера для сырых байт.
2.  **[x] Frame Packer:** Сборщик фреймов из байтов FIFO с таймаутом и CRC.
3.  **[x] Frame Parser & Cmd Handler:** Диспетчеризация и валидация команд.
4.  **[x] Frame Sender (TX):** Формирование бинарных ответов (ACK/NACK/DATA).
5.  **[x] ADC Driver (Industrial):** Чтение температуры чипа через ADC1 + DMA1 + EMA фильтр.
6.  **[ ] OneWire & DS18B20:** Драйвер внешнего датчика.
7.  **[ ] Autonomous Thermostat:** Логика поддержания температуры (Master STM32).

**Архитектурное решение (ADC v1.0):**
*   **Транспорт:** DMA1 Channel 1 в циклическом режиме (Circular) для нулевой нагрузки на CPU.
*   **Точность:** Sample Time = 239.5 циклов (максимальный для компенсации импеданса датчика).
*   **Фильтрация:** Экспоненциальное скользящее среднее (EMA) с коэффициентом Alpha = 0.1.
*   **Telemetry Sync:** 
    *   `USER_DATA` (0x03): Основной Heartbeat (30с) + Event-driven (Delta T > 0.5°C).
    *   `HEALTH_DATA` (0x04): Только по событию (смена Relay, Error) + контрольный Slow Heartbeat (5 мин).

### Фаза 1.3: Сервисный Рефакторинг (ВЫПОЛНЕНО)
