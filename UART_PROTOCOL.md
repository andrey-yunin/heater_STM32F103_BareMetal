# Спецификация протокола UART v1.2: Smart Heater

## 1. Формат кадра (Frame Structure)
| Поле | Размер (байт) | Значение | Описание |
| :--- | :--- | :--- | :--- |
| **START** | 1 | `0x24` (`$`) | Маркер начала пакета. |
| **CMD** | 1 | `0x00..0xFF` | Код команды. |
| **LEN** | 1 | `0x00..0x10` | Длина поля DATA. |
| **DATA** | LEN | `...` | Тело сообщения (Big-Endian). |
| **CRC8** | 1 | `0x00..0xFF` | XOR: CMD ^ LEN ^ DATA_0 ^ ... ^ DATA_N. |

---

## 2. Таблица команд

### Запросы (Host ESP32 -> Controller STM32)
| CMD | Название | LEN | Описание |
| :--- | :--- | :--- | :--- |
| **0x10** | **PING** | 0 | Проверка связи. Ответ: ACK (0x01). |
| **0x11** | **GET_USER_DATA**| 0 | Запрос данных для экрана. Ответ: USER_DATA (0x03). |
| **0x12** | **GET_HEALTH** | 0 | Запрос диагностики. Ответ: HEALTH_DATA (0x04). |
| **0x13** | **SET_TARGET** | 2 | Установка целевой температуры (int16). Ответ: ACK. |
| **0x14** | **RELAY_CTRL** | 1 | Ручное упр. реле (0-Off, 1-On). Ответ: ACK. |

### Ответы (Controller STM32 -> Host ESP32)
| CMD | Название | LEN | DATA (наполнение) |
| :--- | :--- | :--- | :--- |
| **0x01** | **ACK** | 1 | `u8`: ID выполненной команды. |
| **0x02** | **NACK** | 1 | `u8`: Код ошибки (0x01-CRC, 0x02-ID). |
| **0x03** | **USER_DATA** | 4 | `int16` (T_curr) + `int16` (T_target). |
| **0x04** | **HEALTH_DATA** | 7 | `u8`(SensStat) + `u8`(Relay) + `int8`(MCU_T) + `u32`(Uptime). |
| **0x05** | **EMERGENCY** | 1 | `u8`: Битовая маска аварии (0x01-Sensor, 0x02-Overheat). |

---

## 3. Детализация данных (Data Types)
*   **Температура (int16):** `Значение * 10`. Пример: 22.5°C = 225.
*   **Uptime (u32):** Секунды с момента старта.
*   **Sensor Status:** 0-OK, 1-No Signal, 2-Short Circuit.
*   **MCU Temp (int8):** Температура кристалла STM32 в целых градусах.
*   **Emergency (инициативный):** STM32 шлет этот пакет сам при обнаружении аварии.

---

## 4. Регламент обмена (Industrial Timings)

### 4.1. Heartbeat (Пульс системы)
*   **USER_DATA (0x03)**: Отправляется каждые **30 секунд**. Служит для синхронизации уставки и текущей температуры.
*   **HEALTH_DATA (0x04)**: Плановая отправка раз в **5 минут**. 

### 4.2. Реактивность (Event-driven)
*   **При изменении Relay**: Пакет `0x04` отправляется **немедленно**. Максимальная задержка от физического щелчка до пакета в UART — 1 мс.
*   **При изменении Target**: Ответ `ACK (0x01)` отправляется немедленно. 

### 4.3. Обработка ошибок
*   При потере связи с DS18B20 в поле температуры передается значение **`-999`**.
*   Бридж обязан трактовать `-999` как аварию и переходить в режим красной индикации.
*   STM32 при получении `-999` автоматически переводит реле в состояние **OFF**.
